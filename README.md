## 修改说明

### 安全检查逻辑

1. 页面`初始化`，使用固定的`初始数据`，初始化页面；
2. 获取检查信息，这里有三个部分：`检查`，`核对`，`确认`；
3. 如果有这三部分中的至少一个的数据，将信息重新填入页面，但是页面不能出现`闪动`，`刷新效果`；
4. 表格内容压缩为：`检查项目`，`要求`，`状态`，`检查进度`；
5. `状态`有三种：是-Y、否-Y、NA-NA；
6. `检查进度`分四步：第一步：还没有检查，显示为`待说明`，此时点击这一个按钮，进入说明添加页面，添加说明，确认返回，如果说明内容不为空，则将按钮的内容改为`已说明`，点击提交，进入第二步；
7. 第二步：检查完成，待核对。此时这一项显示：有检查内容的显示为`待解决`，没有检查内容的显示`已完成`。`待解决`可以点击进入，添加解决方法，点击提交返回，`待解决`变为`已解决`。`已完成`不可点击进入，并提示`没有要解决的问题`。点击提交，进入第三步；
8. 第三步：核对完成，待确认。此时这一项显示：有核对内容的显示为`待确认`，没有的显示`已完成`。`待确认`可以点击进入，添加确认信息，点击提交返回，`待确认`变为`已确认`。`已完成`不可点击进入，并提示`没有需要确认的信息`。点击提交，进入第四步；
9. 第四步：确认完成，待修改。有确认信息的显示`查看`，没有的显示`已完成`；
10. 第五步：如果修改了检查项，`已完成`变为`已说明`，提交之后，变为`待核对`
